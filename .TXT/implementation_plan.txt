KEY TECHNICAL DECISIONS
────────────────────────────────────────────────────────────────────────────────

  Decision                    Rationale
  ──────────────────────────  ──────────────────────────────────────────────────
  Google ADK as agent         First-class support for Gemini function calling;
  framework                   purpose-built for multi-turn tool-use agents.

  Google Gemini 1.5 Pro/Flash Native ADK integration; instruction-following and
  as LLM                      function calling quality matches requirements.
                              Flash for latency-sensitive turns, Pro for digests.

  Google BigQuery as          Fivetran's native destination; scalable SQL at
  data warehouse              cost proportional to bytes scanned; no ops burden.

  Fivetran for data           Managed, reliable ELT with a pre-built MCP server;
  ingestion                   eliminates custom connector maintenance entirely.

  Fivetran MCP server for     Agent-native protocol; avoids building a custom
  connector control           REST client for Fivetran's management API.

  Parameterised BigQuery      Prevents SQL injection from user queries; required
  queries only                by the system prompt to force time-filter discipline.

  No individual developer     Explicit PRD non-goal; avoids PII concerns and
  metrics exposed             misuse of performance data; GitHub usernames hashed.

  In-memory conversation      Sufficient for hackathon scope; 10-turn window
  history (10 turns)          keeps token costs bounded. No persistent session DB.

  Cloud Run for deployment    Serverless; scale-to-zero minimises cost when idle;
  (optional)                  Cloud Scheduler handles the 08:00 cron trigger.

  YAML + env vars for config  No secrets in source code; portable across dev,
                              staging, and prod environments without code changes.
