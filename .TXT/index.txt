    create_runner() wraps it with InMemorySessionService (async) + Runner.
    Model configurable via GEMINI_MODEL env var (default: gemini-1.5-flash).
    Session history: per session_id, unlimited turns (ADK-managed).

  [DONE]  Gemini system prompt with 5 hard rules
    agent/system_prompt.py: SYSTEM_PROMPT constant encodes:
    (1) Always call check_connector_status before any data query.
    (2) Prepend staleness warning if any connector >4h old.
    (3) Never present individual developer performance — aggregate only.
    (4) When anomalies detected, include severity + recommended action.
    (5) All SQL queries must include a time filter.

  ── Fivetran Layer ────────────────────────────────────────────────────────────

  [DONE]  check_connector_status(connector_id) → dict
    tools/fivetran_tools.py. Calls Fivetran REST API GET /connectors/{id}.
    Returns connector_id, name, service, status, last_successful_sync_at, is_stale.
    is_stale = True if last sync > 4 hours ago.
    On HTTP error or network failure: returns { status: "unreachable", is_stale: True }.
    MOCK_MODE: returns from mock/fixtures.py CONNECTOR_STATUS.

  [DONE]  trigger_sync(connector_id) → dict
    tools/fivetran_tools.py. POST /connectors/{id}/force, then polls every 30s.
    Timeout: 300s. Returns { success, sync_completed_at } or { success: False, error }.
    MOCK_MODE: returns immediate success with current timestamp.

  [DONE]  list_connectors() → list[dict]
    tools/fivetran_tools.py. GET /connectors, returns all connectors with sync status.
    MOCK_MODE: returns CONNECTORS_LIST fixture (2 connectors: GitHub + Jira).

  ── BigQuery Tool Layer ───────────────────────────────────────────────────────

  [DONE]  BigQuery schema and view DDL
    schema/tables.sql: CREATE TABLE IF NOT EXISTS for agent_digests,
    agent_anomalies, agent_logs. Uses {project}/{dataset} substitution.
    schema/views.sql: CREATE OR REPLACE VIEW for all 5 views.
    schema/deploy_schema.py: reads both SQL files, substitutes placeholders,
    executes via BQ client. Creates dataset if missing. Idempotent.

  [DONE]  query_open_prs(min_age_days=3) → list[dict]
    tools/bq_tools.py. Queries v_open_prs_aged with parameterised min_age_days.
    Returns: pr_number, title, author_login, repo_name, age_days, age_bucket,
    last_reviewer, last_review_state, last_reviewed_at, created_at.

  [DONE]  query_commit_velocity(days=7) → list[dict]
    tools/bq_tools.py. Queries v_commit_velocity_daily for last N days.
    Returns: commit_date, repo_name, commit_count, additions_sum, deletions_sum.

  [DONE]  query_stale_tickets(status, min_age_days=14) → list[dict]
    tools/bq_tools.py. Queries v_ticket_age_by_status with parameterised status +
    min_age_days. Returns: key, summary, status, issue_type, priority,
    assignee_id, sprint_id, story_points, age_days, days_since_update.

  [DONE]  query_sprint_health() → dict
    tools/bq_tools.py. Queries v_sprint_burndown for active sprint.
    Returns: sprint_name, start_date, end_date, total_points, points_completed,
    points_remaining, tickets_done, tickets_blocked, days_remaining, pct_complete,
    expected_pct_complete, is_on_track.

  [DONE]  generate_trend_report(window_days=30) → dict
    tools/bq_tools.py. Aggregates commit velocity, stale PRs, ticket breakdown
    over window_days. Returns structured summary dict.

  [DONE]  detect_anomalies() → list[dict]
    tools/bq_tools.py. Queries v_metric_baselines for rolling mean + stddev.
    z_score = (value - mean) / stddev. Flags |z| > 2. Severity: low (2-3),
    medium (3-4), high (>4). Writes each anomaly to agent_anomalies via
    _write_anomaly(). Returns list of anomaly dicts.

  [DONE]  write_digest(digest_data) → str (UUID)
    tools/bq_tools.py. Inserts one row to agent_digests. Generates UUID.
    anomaly_summary serialised to JSON string. Returns digest_id.

  [DONE]  write_agent_log() (internal, not ADK tool)
    tools/bq_tools.py. Called by interface layer after each agent turn.
    Writes to agent_logs. Silent on failure (logging only).

  5. DATABASE SCHEMA
────────────────────────────────────────────────────────────────────────────────

  Dataset: devpulse (Google BigQuery)
  Tables prefixed github_ / jira_ are Fivetran-managed (read-only).
  Tables prefixed agent_ are agent-managed (read-write).

  ── Fivetran-Managed Tables (read-only) ──────────────────────────────────────

  TABLE: github_pull_requests
    id, repo_name, number, title, state, author_login, created_at, updated_at,
    merged_at, closed_at, review_requested_at, _fivetran_synced

  TABLE: github_commits
    sha, repo_name, author_login, committed_at, additions, deletions,
    message_summary, _fivetran_synced

  TABLE: github_reviews
    id, pull_request_id, reviewer_login, state, submitted_at, _fivetran_synced

  TABLE: jira_issues
    id, key, project_key, summary, issue_type, status, priority, assignee_id,
    reporter_id, created_at, updated_at, resolved_at, sprint_id, story_points,
    _fivetran_synced

  TABLE: jira_sprints
    id, name, board_id, state, start_date, end_date, completed_date,
    _fivetran_synced

  ── Agent-Managed Tables (read-write) ────────────────────────────────────────

  TABLE: agent_digests
    digest_id (UUID PK), generated_at, digest_date, open_pr_count,
    stale_pr_count, commit_count_7d, commit_count_prev_7d, anomaly_count,
    anomaly_summary (JSON), sprint_on_track, narrative, model_used

  TABLE: agent_anomalies
    anomaly_id (UUID PK), detected_at, metric_name, metric_value,
    baseline_mean, baseline_stddev, z_score, severity, entity_type,
    entity_id, resolved_at

  TABLE: agent_logs
    log_id (UUID PK), session_id, logged_at, user_query, tools_called (JSON),
    bq_bytes_processed, latency_ms, model_used, response_preview

  ── BigQuery Views ────────────────────────────────────────────────────────────

  VIEW: v_open_prs_aged         → github_pull_requests JOIN github_reviews
  VIEW: v_commit_velocity_daily → github_commits (GROUP BY date, repo)
  VIEW: v_ticket_age_by_status  → jira_issues (WHERE resolved_at IS NULL)
  VIEW: v_sprint_burndown       → jira_issues JOIN jira_sprints (active sprint)
  VIEW: v_metric_baselines      → 30-day AVG + STDDEV per metric name

  6. API ROUTES
────────────────────────────────────────────────────────────────────────────────

  All routes live in interface/api.py. Server: uvicorn interface.api:app.
  Swagger UI: http://localhost:8000/docs

  GET   /health
    Returns { status, mock_mode, model }. No auth. Used for health checks.

  POST  /query
    Body: { "question": str, "session_id": str (optional) }
    Returns: { "answer": str, "session_id": str, "latency_ms": int }
    Runs a full ADK agent turn. session_id maintained across calls for history.
    Non-obvious: if session_id omitted, creates a new stateless session per call.

  GET   /connectors/status
    Returns: { "github": {…connector dict…}, "jira": {…connector dict…} }
    Calls check_connector_status for both configured connectors.

  POST  /sync/trigger?connector=github|jira
    Triggers Fivetran ad-hoc sync. Polls until complete (up to 5 min).
    Returns: { connector_id, success, sync_completed_at } or { error }.

  GET   /prs/open?min_age_days=3
    Returns: { "prs": [list], "min_age_days": int }
    Calls query_open_prs directly (bypasses agent for raw data access).

  GET   /sprint/health
    Returns active sprint burn-down dict including is_on_track flag.

  GET   /anomalies
    Runs detect_anomalies() and returns { "anomalies": [list], "count": int }.

  POST  /digest/generate
    Triggers the full daily_digest.run_digest() flow.
    Returns: { "digest_id": str, "message": "Digest generated successfully." }

  9. ARCHITECTURE NOTES
────────────────────────────────────────────────────────────────────────────────

  ── AI PIPELINE (live query) ─────────────────────────────────────────────────

  POST /query  →  FastAPI (interface/api.py)
    → ADK Runner.run_async(session_id, question)
    → Gemini reads SYSTEM_PROMPT → decides tool calls
    → check_connector_status (GitHub) via Fivetran REST API
    → check_connector_status (Jira) via Fivetran REST API
    → If stale: prepend ⚠️ warning to response
    → Route to 1..N BigQuery tools (all parameterised)
    → BigQuery executes queries, returns structured rows
    → Gemini synthesises rows → natural-language answer
    → If insight detected: write_digest → agent_digests
    → Return answer, session_id, latency_ms
    → write_agent_log → agent_logs (fire-and-forget)

  ── DAILY DIGEST FLOW ────────────────────────────────────────────────────────

  python jobs/daily_digest.py  (or POST /digest/generate)
    → list_connectors → detect_anomalies → query_open_prs(0)
    → query_commit_velocity(7) → query_commit_velocity(14)
    → query_sprint_health → build context dict
    → google.generativeai.GenerativeModel.generate_content (direct call)
    → fallback: _template_narrative if Gemini fails
    → write_digest → agent_digests table
    → post_to_slack (no-op if SLACK_WEBHOOK_URL unset)

  ── MOCK MODE ────────────────────────────────────────────────────────────────

  MOCK_MODE=true in .env activates fixture short-circuits in every tool.
  Each function checks os.getenv("MOCK_MODE") at the top and returns
  from mock/fixtures.py if true. The ADK agent still runs; only the data
  layer is mocked. Requires GOOGLE_API_KEY for the Gemini call.

  ── SECURITY MODEL ───────────────────────────────────────────────────────────

  All secrets in .env; never in source. .gitignore must include .env.
  FIVETRAN_API_KEY scoped to read + trigger-sync only.
  BigQuery service account scoped to devpulse dataset only.
  Individual developer metrics never exposed — aggregate only (system prompt rule).
  All BigQuery queries parameterised (no SQL injection vector from user input).
  CORS open in development; tighten allow_origins in production deployment.
